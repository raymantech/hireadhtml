# 开发建议
## 1.文件的命名
在设计每个新题型页面的时候，新建三个文件：
- example-new.html
- example-new.css
- example-new.js

## 2.开发规范
- **请不要在原有代码上修改**，而是新建文件
- html页面中请使用有意义的符合页面结构的id,class名称
- 专注页面静态样式，js能不要则不要,如果确实需要，逻辑尽可能简洁并写注释
- 尽量少依赖第三方样式和类库，如果确实需要，请使用成熟的，文档齐全的框架
- 易于后端开发人员的理解和整合
